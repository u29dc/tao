# Tao Rename Manifest (obs -> tao)

## 1) Objective

Rename project identity from `obs` to `tao` across source, build scripts, package metadata, binaries, runtime paths, and repository coordinates without altering external product names (for example Obsidian references).

## 2) Rename Classes

1. Repository and remote identity:
- `u29dc/obs` -> `u29dc/tao`
- `git@github.com:u29dc/obs.git` -> `git@github.com:u29dc/tao.git`

2. Workspace/package identity:
- crate names `obs-*` -> `tao-*`
- package name `obs` -> `tao`
- binary names `obs`, `tao-tui`, `tao-sdk-bridge` -> `tao`, `tao-tui`, `tao-sdk-bridge`

3. Directory identity:
- `apps/tao-macos` -> `apps/tao-macos`
- Rust crate directories `crates/tao-*` -> `crates/tao-*`

4. Runtime/config identity:
- env vars `OBS_*` -> `TAO_*`
- default sqlite path `.obs.sqlite` -> `.tao.sqlite`
- install path `${HOME}/.tools/obs` -> `${HOME}/.tools/tao`

5. Source symbols and labels:
- Swift/Rust identifiers, user-facing command names, docs references

## 3) Exclusions and Guardrails

Do not rename:

1. `Obsidian` product name in compatibility/spec docs.
2. External source URLs that include `obsidian` or third-party names.
3. Historical release notes where string replacement would corrupt chronology.
4. `.git` metadata internals.
5. Binary blobs and generated lock hashes unless regenerated by tooling.

## 4) Deterministic Sweep Order

1. Rename repository remote and origin URLs.
2. Rename top-level package metadata and scripts.
3. Rename workspace members and crate directories.
4. Update all Rust imports and crate references.
5. Rename Swift package/module/app and update imports.
6. Rename scripts/env/runtime defaults.
7. Run stale-reference sweep with explicit allowlist.

## 5) Validation Queries

Run after each major step:

1. `rg -n "\bobs\b|obs-" --glob '!target/**' --glob '!.git/**'`
2. `rg -n "OBS_|\.obs\.sqlite|/obs\b" --glob '!target/**' --glob '!.git/**'`
3. `cargo metadata --no-deps`
4. `bun run util:check`
5. `swift test --package-path apps/tao-macos`

## 6) Allowed Residual References

The following are allowed after completion:

1. Obsidian documentation content and URLs.
2. Historical progress entries that explicitly mention legacy names.
3. Git remote reflog/history text.

## 7) Completion Gate

Rename is complete only when:

1. Commands/binaries/package names are `tao`.
2. Runtime defaults are `TAO_*` and `.tao.sqlite`.
3. App package path and module naming are `tao`-prefixed.
4. No non-allowlisted `obs` identity references remain in active code paths.
